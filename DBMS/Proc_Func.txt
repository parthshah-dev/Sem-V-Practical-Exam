-- Create Table
CREATE TABLE Student (
    RollNo NUMBER PRIMARY KEY,
    Name   VARCHAR2(30),
    Marks  NUMBER,
    Category VARCHAR2(20)
);

-- ==========================================================
-- Stored Function: Returns Category based on Marks
-- ==========================================================
CREATE OR REPLACE FUNCTION GetCategory(v_marks NUMBER)
RETURN VARCHAR2
IS
    v_category VARCHAR2(20);
BEGIN
    IF v_marks >= 75 THEN
        v_category := 'Distinction';
    ELSIF v_marks >= 60 THEN
        v_category := 'First Class';
    ELSIF v_marks >= 50 THEN
        v_category := 'Second Class';
    ELSE
        v_category := 'Fail';
    END IF;

    RETURN v_category;
END;
/

-- ==========================================================
-- Stored Procedure: Updates Student Category using Function
-- ==========================================================
CREATE OR REPLACE PROCEDURE UpdateCategory IS
BEGIN
    FOR rec IN (SELECT RollNo, Marks FROM Student) LOOP
        UPDATE Student
        SET Category = GetCategory(rec.Marks)
        WHERE RollNo = rec.RollNo;
    END LOOP;

    COMMIT;
    DBMS_OUTPUT.PUT_LINE('Student categories updated successfully.');
END;
/

-- ==========================================================
-- Execute Procedure
-- ==========================================================
BEGIN
    UpdateCategory;
END;
/

-- View Results
SELECT * FROM Student;



Theory: PL/SQL Stored Function and Procedure

A stored function and a stored procedure are reusable subprograms in PL/SQL that help modularize and simplify database operations. They allow logical processing, calculations, and data manipulation directly within the database.

    1. Stored Function:
    A function is a PL/SQL block that performs a specific task and returns a single value.

        - It accepts parameters as input and always returns a result using the RETURN statement.

        - Functions are often used for computations, validations, or deriving values.

        - They can be called inside SQL queries, other PL/SQL blocks, or within stored procedures.

    2. Stored Procedure:
    A procedure is a PL/SQL block that performs one or more actions but does not return a value directly.

        - It can accept parameters and use DML operations like INSERT, UPDATE, and DELETE.

        - Procedures are used for executing business logic, automating repetitive tasks, or performing batch updates.

        - They can be called from an anonymous block or another procedure.

Difference:

A function must return a value, while a procedure may or may not.

Functions are generally used for calculations, whereas procedures are used to execute actions.
